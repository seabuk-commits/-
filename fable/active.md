# Fable 절감형 오케스트레이션 지침

## 최우선 원칙: 총토큰 절감
- 기본 세션은 Haiku 입니다. 대부분의 작업을 메인(Haiku)이 직접 처리합니다.
- 위임은 비용입니다(서브에이전트는 독립 컨텍스트를 별도로 소모). 필요할 때만 위임하십시오.

## 직접 처리(위임 금지)
- 1~2개 파일의 수정·오타·설정/문서 변경·단순 조회·빌드·로그 확인은 메인이 직접 처리합니다.

## 위임 기준(꼭 필요할 때만)
- 대량 파일 탐색/검색으로 컨텍스트가 오염될 상황 -> explorer(Haiku)에게 위임하고 요약만 회수.
- 정말 어려운 근본원인 분석·아키텍처 설계 -> reasoner(Opus, effort 상한 high)에게 위임(예외적, 아껴서).
- Sonnet은 사용하지 않습니다. 승격이 필요하면 Opus(effort high)로만 갑니다.

## 게이트 대응
- 턴당 직접 파일수정은 2개까지입니다. 차단 메시지를 받으면 재시도하지 말고,
  (a) explorer 비서에게 위임하거나 (b) 정말 필요한 수정인지 재검토 후 다음 턴에 진행하십시오.
- 무조건 위임하지 마십시오. 위임 자체가 토큰을 쓰므로, 먼저 "직접 해도 되는가"를 판단합니다.

## 모델 승격은 예외
- Opus 승격은 기본이 아니라 예외입니다. Haiku로 충분한 일은 Haiku로 끝냅니다.
- 승격 시 effort는 high를 상한으로 합니다. max는 사용하지 않습니다.
