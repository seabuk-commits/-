# 전역 사용자 지침 (모든 프로젝트 공통)

이 문서는 어느 프로젝트에서 작업하든 항상 적용되는 사용자의 표준 작업 방식이다.

## 1. 확인하고 진행하라

설계/범위/우선순위 같은 판단은 스스로 내려서 구체적인 계획으로 정리하되, 실행에 옮기기
전에 "이렇게 진행해도 될까요?" 같은 한 줄 확인을 거친다. 사소하고 되돌리기 쉬운 일까지
매번 세세하게 되묻지는 않지만, 파일 수정·커밋·설정 변경처럼 흔적이 남는 작업은 실행 전에
사용자가 "네/진행하세요" 정도로 답할 수 있는 짧은 확인 문구를 반드시 보여준다.

## 2. 도구 실행도 확인 후 진행

도구 승인 프롬프트가 뜨면 정상적으로 사용자의 승인을 받는다. 실행이 막히거나 실패해도
조용히 멈추지 말고, 원인을 보고하고 다음 단계를 제안한다.

## 3. 한국어로 응답

대화의 모든 텍스트(설명, 답변)는 한국어로 작성한다. 도구 호출 설명/승인 프롬프트에 표시되는
텍스트도 한국어로 쓴다.

## 4. 토큰 사용을 아낀다

불필요한 재탐색·재확인·중복 파일 읽기를 피하고, 도구 호출을 배치로 묶을 수 있으면 묶는다.
꼭 필요하지 않으면 서브에이전트를 새로 띄우지 않는다. 크고 무거운 원본을 통째로 컨텍스트에
넣기보다, 가능하면 요약된 결과만 받아서 쓴다.

## 5. 다른 컴퓨터 동기화 로그 유지

이 전역 CLAUDE.md(`~/.claude/CLAUDE.md`)와 메모리 폴더(`~/.claude/projects/<프로젝트해시>/memory/`)는
계정명이 컴퓨터마다 달라(회사=user, 집=박성용) 구글 드라이브 자동 동기화 대상 밖에 있다.
그래서 **이 파일을 수정하거나 메모리 파일을 새로 쓰거나 고칠 때마다**,
`G:\내 드라이브\claude-settings\동기화.md`에 날짜·파일명·한 줄 요약을 추가한다. 다른
컴퓨터에서 세션을 열었을 때 이 로그 하나만 보고 무엇이 바뀌었는지 파악해 로컬에 반영할 수
있게 하기 위함이다. 프로젝트 CLAUDE.md, `.claude/agents/`, `.claude/scripts/`처럼 이미
구글 드라이브 폴더 안에 있는 파일은 자동 동기화되므로 기록 대상이 아니다.
